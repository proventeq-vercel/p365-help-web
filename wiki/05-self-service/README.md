# Self Service

The **Self Service** feature enables users to independently raise, track, and manage requests through a structured and governed workflow, reducing reliance on administrators while ensuring compliance with organisational policies.

Users can initiate requests using **Templates**, which provide predefined and standardised request types to ensure consistency and accuracy. The **New Request** option allows users to submit requests based on these templates, while **Request History** provides visibility into previously submitted requests along with their current status and outcomes. The **Approvals** section supports controlled decision-making by allowing designated approvers to review, approve, or reject requests assigned to them.

Together, these capabilities streamline operational processes, improve turnaround times, and maintain clear accountability across self-service activities.

## Self-Service Workflow

1. Open **Self Service** and select or create a **Template** (predefined request type).
2. Choose **New Request**, complete the required details, and **submit**.
3. Track progress in **Request History** (status, updates, outcome).
4. If approval is required, approvers review and act in **Approvals** (approve/reject).
